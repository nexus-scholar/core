## 1. What This Project Is

`nexus-php` is a **PHP 8.3 library** for Systematic Literature Reviews (SLR). It:

- Searches OpenAlex, Crossref, arXiv, Semantic Scholar, PubMed, DOAJ, and IEEE Xplore through a uniform provider model.
- Deduplicates overlapping results into stable clusters with elected representatives.
- Expands a corpus by forward/backward snowballing.
- Builds citation, co-citation, and bibliographic-coupling graphs with PageRank, k-core, and BFS path metrics.
- Exports bibliographies (BibTeX, RIS, CSV, JSON, JSONL) and graphs (GEXF, GraphML, Cytoscape).
- Retrieves PDFs from multiple sources (arXiv, OpenAlex, Semantic Scholar, direct link).
- Integrates with Laravel as a thin optional adapter — the core domain has zero `Illuminate\*` imports.

## 3. Architecture Rules (Absolute — Never Break)

### 3.1 No framework leakage into the domain
`src/Shared`, `src/Search`, `src/Deduplication`, `src/CitationNetwork`, and `src/Dissemination` **must not** import `Illuminate\*`. Violations are architectural failures, not style issues.

### 3.2 No infrastructure instantiation in the domain
No `new GuzzleClient()`, `new Redis()`, `new PDO()` inside domain or application services. Everything external arrives through **ports**.

### 3.3 Rate limiting is mandatory and structural
Every provider adapter **must** call `$this->rateLimiter->waitForToken()` before every HTTP request. The original package had a rate limiter that was never wired — that was the most critical bug. The redesign makes it impossible to skip: `BaseProviderAdapter::request()` centralises it. Any adapter that bypasses this is **broken by design**.

### 3.4 Cache key generation belongs to `SearchQuery`
`SearchQuery::cacheKey()` is the **only** place a cache key may be computed. It hashes: term, yearFrom, yearTo, language, maxResults, offset, sorted provider aliases. Never compute a partial cache key elsewhere.

### 3.5 No mutable singleton provider registry
Providers are registered **once** in `NexusServiceProvider::register()` and never mutated per request. The old `clearProviders()`/`registerProvider()` per-request cycle was a concurrency bug. Do not reproduce it.

### 3.6 Raw provider payloads are opt-in only
`ScholarlyWork::$rawData` is `null` unless `SearchQuery::$includeRawData === true`. Never unconditionally store raw provider responses — they can be multi-KB and cause memory exhaustion at scale.

### 3.7 Single source of truth for normalization
DOI normalization lives **only** in `WorkId::normalize()`. No other class may implement it. All other code uses `WorkId` to normalize.

### 3.8 Graph algorithms must be scalable
Co-citation and bibliographic coupling **must** use inverted indexes — never O(n²) nested pairwise loops. Implementations: `InvertedIndexCoCitation`, `InvertedIndexBibCoupling`.

### 3.9 SSL verification via maintained CA bundle
Use `composer/ca-bundle` at runtime. **Never** commit a `cacert.pem` file.

### 3.10 No personal data in source control
`mailto` values for OpenAlex/Crossref polite-pool must come from `env('NEXUS_MAILTO')`. Never hardcode email addresses in source.

## 5. Layer Responsibilities

### Domain
Business concepts, invariants, language. No transport, no framework, no persistence details.

### Application
Coordinates use cases. Calls ports. Orchestrates domain objects. No HTTP parsing, no SQL, no framework event logic.

### Infrastructure
Implements ports: provider adapters, storage, serializers, persistence, rate limiting, HTTP client wrappers.

### Laravel
Bridges host app and package: service container bindings, queue jobs, Eloquent models, Artisan commands, published config, event bridges. Contains no domain logic.

## 6. Key Classes & Their Contracts

### 6.1 Shared Kernel

**`WorkIdNamespace` (enum)** — `doi | arxiv | openalex | s2 | pubmed | ieee | doaj`. Exhaustive list. Do not add values without a globally meaningful identifier space.

**`WorkId`** — immutable, always normalized in constructor.
- DOI: strip `https://doi.org/`, `http://dx.doi.org/`, `doi:` → `strtolower()` → `trim()`
- ARXIV: strip `arxiv:` prefix → `strtolower()` → `trim()`
- Others: `strtolower()` → `trim()`
- `fromString('doi:10.x')` must equal `new WorkId(DOI, 'https://doi.org/10.x')`

**`WorkIdSet`** — immutable. `add()` and `merge()` return new instances.
Primary ID precedence: DOI → OPENALEX → S2 → ARXIV → PUBMED → IEEE → DOAJ.

**`Author`** — `familyName`, `givenName?`, `orcid?`, `normalizedFullName?`. `isSamePerson()` checks ORCID first, then normalized name.

### 6.2 Search Domain

**`SearchQuery`** — generates its own ID via `'Q' . bin2hex(random_bytes(5))`. **Never `uniqid()`**. `cacheKey(array $sortedProviderAliases)` hashes all dimensions.

**`ScholarlyWork`** — private constructor, use `::reconstitute()` static factory. `isSameWorkAs()` delegates entirely to `WorkIdSet::hasOverlapWith()`. `mergeWith()` never overwrites existing fields.

**`CorpusSlice`** — aggregate root. `addWork()` merges duplicates rather than inserting; `merge()` returns a new instance.

### 6.3 Search Ports

**`AcademicProviderPort`** — `alias()`, `search(SearchQuery): ScholarlyWork[]`, `fetchById(WorkId): ?ScholarlyWork`, `supports(WorkIdNamespace): bool`.

**`RateLimiterPort`** — `waitForToken()` (blocking), `tryConsume()` (non-blocking), `ratePerSecond()`. **This is the single most important contract in the system.**

**`SearchCachePort`** — `get/put/has/invalidateAll()`. `invalidateAll()` bumps a version counter prefix — never relies on tag flushing (the old `tags()->flush()` was a silent no-op).

### 6.4 Search Infrastructure

**`BaseProviderAdapter`** — abstract. `request()` is `final`: calls `waitForToken()` then HTTP get. Retry: max 3 attempts, exponential backoff 1s/2s/4s. Retry on 429/5xx/timeout. Do NOT retry on 404/401/403.

**`TokenBucketRateLimiter`** — uses `hrtime(true)`, not `microtime()`.

**`GuzzleHttpClient`** — uses `composer/ca-bundle` for SSL. Never commits CA file.

### 6.5 Deduplication

**Policy pipeline order** (cheapest/most reliable first):
1. `DoiMatchPolicy` — O(n) index, confidence 1.0
2. `NamespaceMatchPolicy` (ARXIV, OPENALEX, S2, PUBMED) — O(n) index, confidence 1.0
3. `TitleFuzzyPolicy` — sorted-list adjacent comparison (not O(n²)), Unicode-safe via `mb_strtolower` + `iconv` transliteration + `mb_str_split` Levenshtein. Default threshold: 92 (not 97).
4. `FingerprintPolicy` — md5(title[0:50] + ':' + firstAuthorFamily + ':' + year), O(n).

**`UnionFind`** — used by `DeduplicateCorpusHandler`. Groups transitively connected works.

**`CompletenessElectionPolicy`** — score = `completenessScore()` + `providerPriority[$sourceProvider]`. Provider priority is **injected from config**, never hardcoded.

**`TitleNormalizer`** — `mb_strtolower` → strip HTML entities → `iconv` diacritics → strip non-alphanumeric → collapse spaces → `trim()`. **Never `strlen()` or `levenshtein()` on raw UTF-8.**

### 6.6 Citation Network

**`CitationGraph`** invariant: `recordCitation($citing, $cited)` throws `WorkNotInGraph` if `$citing` not present. External citations (target not in graph) are allowed.

**`SnowballConfig`** — depth 1–5 (throws on 0 or >5). `forward`, `backward`, `maxCitations`, `maxReferences`.

**`InvertedIndexCoCitation`** — O(n·k) not O(n²).

**`InvertedIndexBibCoupling`** — O(n·k) not O(n²).

**`PageRankCalculator`** — damping 0.85, max 100 iterations, convergence 1e-6.

### 6.7 Dissemination

**Bibliography formats**: BibTeX, RIS, CSV, JSON, JSONL.
**Graph formats**: GEXF 1.3 (DOMDocument), GraphML, Cytoscape JSON.
**Serializers are pure** — no I/O, no side effects. Storage handled by `FileStoragePort`.
**PDF sources** (tried in order): arXiv → OpenAlex → SemanticScholar → Direct. All attempts are logged individually.

### 6.8 Laravel Layer

**`NexusServiceProvider`** — registers ALL providers once at boot. Uses `tagged()` bindings for collection injection.

**Jobs**: `SearchJob`, `SnowballJob`, `DeduplicateCorpusJob`, `RetrieveFullTextJob`. All call application services — no business logic inside jobs.

**Eloquent models** are in `src/Laravel/Models/` and are infrastructure projections. Domain code **never** imports them.

**AI Tools** (`LiteratureSearchTool`, `SnowballTool`, `CitationGraphTool`) wrap use cases. They contain no domain logic.

## 7. Persistence Model

### Philosophy
1. **Immutable findings, mutable decisions.** `ScholarlyWork` rows are never overwritten. Dedup clusters, screening decisions, and PDF fetch attempts are separate append tables.
2. **Cluster provenance.** `document_clusters` stores strategy + threshold + max_year_gap so reruns are comparable.
3. **Query provenance.** Every work traceable to exact query + provider for PRISMA reporting.

### Migration order (respect FK dependencies)
1. `slr_projects`
2. `authors`
3. `documents` (no `rawData` column, no `provider` column)
4. `document_external_ids` (UNIQUE on `doi` — last-line dedup defense)
5. `document_providers`
6. `document_authors`
7. `search_queries`
8. `search_query_providers`
9. `query_documents`
10. `document_clusters`
11. `cluster_members`
12. `screening_decisions`
13. `pdf_fetches`
14. `citation_graphs`
15. `citation_edges`

### Key schema decisions
- ULIDs everywhere (not auto-increment) — globally unique, URL-safe, lexicographically sortable.
- No `rawData` / `provider` columns on `documents` table.
- `screening_decisions` is append-only per stage, supporting conflict detection.
- `pdf_fetches` logs all attempts per source (not just final success).
- `citation_graphs.metadata` JSON stores computed metrics (PageRank, k-core) — avoids a separate metrics table.

## 8. TDD Strategy

Write tests in this order:

1. **Value objects** — `WorkId`, `WorkIdSet`, `SearchQuery`, `YearRange`, `AuthorList` (fast, define the language).
2. **Entities & aggregates** — `ScholarlyWork`, `CorpusSlice`, `DedupCluster`, `CitationGraph`, `SnowballRound`.
3. **Application services** — mock ports, test use cases (reads like a specification).
4. **Adapter integration tests** — recorded HTTP fixtures, never live APIs in CI.
5. **Laravel feature tests** — service provider bindings, job dispatches, DB projections.

### Test naming style
Use behavior-focused `it_*` names:
- ✅ `it_normalizes_doi_prefixes`
- ✅ `it_clusters_two_works_with_the_same_doi`
- ❌ `testQuery`, `testProvider`, `it_works`

### Golden rule
If a test requires a real network, a real DB, and a queue worker just to validate a domain rule, **the design is wrong**.

Ask these questions before every edit:

1. Which bounded context **owns** this concept?
2. Is this **domain** behavior or **infrastructure** behavior?
3. Is there already a **port** for this?
4. Will this introduce `Illuminate\*` into the core?
5. Will this **duplicate** normalization or matching logic?
6. Can this be proven with a **fast, isolated test**?

## 10. Specific Action Rules

### If adding a provider
- Implement `AcademicProviderPort`
- Extend `BaseProviderAdapter` (rate limiting is inherited via `request()`)
- Add recorded HTTP fixtures for integration tests
- Define supported `WorkIdNamespace` values
- Map raw payloads → `ScholarlyWork` in `normalize()`
- Decide if it also implements `SnowballingProviderPort`

### If adding a dedup rule
- Define its `DuplicateReason` and confidence range
- Add tests for false positives **and** false negatives
- Ensure it composes with earlier (higher-priority) rules
- Clarify whether it changes representative election

### If adding a graph feature
- State algorithmic complexity
- Explain how it scales to 10k+ works
- Do not mutate persistence projections directly
- Add complexity-aware fixtures or benchmark tests

### If adding persistence
- Justify whether it stores **observations** (immutable) or **decisions** (mutable)
- Preserve provenance
- Avoid denormalizing provider ownership into central work rows
- Avoid storing raw provider payloads by default

### If unsure
Prefer: clearer domain names · smaller interfaces · immutable value objects · explicit configuration · pure serializers · deterministic tests.

## 11. Known Bugs from Old Package (Do Not Repeat)

| # | Lesson |
|---|---|
| 1 | Rate limiter existed but was never wired → all API limits were fiction |
| 2 | `AggressiveStrategy` documented but not implemented → erodes trust |
| 3 | `clearProviders()`/`registerProvider()` on shared singleton → concurrency bug |
| 4 | Cache key omitted `language`, `maxResults`, `offset` → stale results |
| 5 | `cacert.pem` committed to repo → staleness + bloat |
| 6 | Snowball service hardcoded dedup strategy → unconfigurable |
| 7 | `fuseDocuments()` ignored configured `providerPriority` → silent misconfiguration |
| 8 | O(n²) co-citation & bib-coupling → scales to ~1k works only |
| 9 | `searchAsync()` accessed non-existent `->id` on `PendingDispatch` → silent runtime error |
| 10 | `rawData` stored on every work unconditionally → memory exhaustion |
| 11 | Personal email hardcoded in `config.yml` → committed to source control |
| 12 | `levenshtein()`/`strlen()` on UTF-8 → wrong fuzzy ratio for non-ASCII titles |
| 13 | `uniqid()` for query IDs → collision-prone in rapid-fire use |
| 14 | DOI normalization duplicated in 3 places → divergence risk |
| 15 | `clearAllCache()` flushed an untagged namespace → always a no-op |

## 12. Provider-Specific Notes

| Provider | Rate | Notes |
|---|---|---|
| OpenAlex | 10 req/s | `mailto` from env. Abstract returned as inverted index — must reconstruct string. Paginate if `maxResults > 200`. |
| Crossref | 50 req/s | `mailto` from env for polite pool. `title` is an array, use `[0]`. |
| arXiv | 3 req/s | Atom XML response — parse with SimpleXML. Supports `ARXIV` namespace only (no DOI). |
| Semantic Scholar | 1 req/s (no key), 10 req/s (with key) | S2 paper IDs. Supports snowballing. |
| PubMed | 3 req/s | Requires `NEXUS_PUBMED_KEY`. |
| IEEE Xplore | 1 req/s | Requires `NEXUS_IEEE_KEY`. |
| DOAJ | 2 req/s | No key required. |

Providers supporting `SnowballingProviderPort`: **OpenAlex**, **SemanticScholar**.
ArXiv does **not** support snowballing (no citation data in API).
